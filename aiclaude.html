AI 装配手册
从 Token 到 Agent

一份给初学者的互动式说明书：先动手玩懂每个零件，再看清整条产业链怎么装配起来。

已核对 ✓ 2026.06
把整个 AI 行业想成一条汽车产业链：模型是发动机，产品是整车，Agent 是会自己开车送货的司机，Token 是油，API 是按升加油，会员是包月租车，Coding Plan 是给长途司机办的加油卡，Skill 是司机随车携带的岗位操作规范。下面每个装置，拆解其中一个零件。
SECTION 01 / 总装蓝图

三层结构：发动机 → 整车 → 司机

你看到的所有 AI 产品差异，几乎都能用这张图解释：同一颗发动机，装进不同的车，配不配司机，决定了它"只能聊天"还是"能干完活交付"。

👉 "DeepSeek 只能聊天"= 只有发动机+素车；"ChatGPT 能生成文件"= 整车带了工具；"Claude Code 能写完网页交给你"= 配了司机。

SECTION 02 / 互动实验室

六个装置，玩懂六个名词

每个装置都可以动手操作。数字均为示意性近似，用于建立直觉，精确计费以各官网为准。

装置 01 Token 切词台 TOKEN / 计量单位

模型不按"字"读文章，而是切成一块块 token（约 1 token ≈ 1.5~2 个汉字）。输入和输出都按 token 计费——这就是 AI 世界的"水表电表"。在下面输入任意文字试试：

帮我分析一下MacBookAir在京东618期间的价格竞争力，并给出三条促销建议。
切出 token 数23
这段话作为输入的成本¥0.00015
若模型回你同样长度¥0.00062
这样的问答调用 1 万次¥7.71

※ 切分为演示用近似（每家模型分词器不同）。注意输出单价通常是输入的 3~5 倍——生成比阅读更耗算力。

装置 02 上下文水箱 CONTEXT WINDOW / 工作记忆

上下文窗口是模型一次能"记住"的内容上限，像一个水箱。聊得越久水位越高，装满之后，最早的对话会被挤出去遗忘。拖动滑块模拟一场越聊越长的对话：

93%
💧 水箱已满！最早的 6 轮对话被挤出遗忘——模型已经不记得你开头说过什么了。换更大上下文的模型，或开新对话。

这也解释了为什么"长文档分析"要看上下文：一份 30 万字的报告 ≈ 18 万 token，32K 的模型根本装不下，1M 的模型可以整本吞下。

装置 03 思考模式开关 THINKING / REASONING MODE

很多模型有两种回答方式：直接答，或先在"草稿纸"上推理一遍再答（思考模式）。草稿你看不见，但每个字都算 token、都收钱。拨动开关对比同一个问题的两种回答：

思考模式：开

QUESTION

38 个仓库要分 1200 台 MacBook，按近 30 天销量加权，但 5 个仓库的库容只剩 10 台，怎么分？

ANSWER

🧠 隐藏思考草稿（约 2,400 token，计费但默认不展示）：先算总销量权重…→ 5 个受限仓先按上限 10 台锁定 → 剩余 1150 台在 33 个仓重新归一化权重 → 检查是否产生新的超容 → 迭代两轮收敛…
分两步：① 5 个受限仓先按上限各锁 10 台（共 50 台）；② 剩余 1,150 台在其余 33 个仓按销量权重重新归一化分配，并复检无新超容。（多花了 token，但把"回流重分"算对了）
耗时
~25s
token 费
×8

经验法则：简单问答关思考（快且便宜），数学 / 多约束规划 / 复杂代码开思考（贵几倍但不容易错）。这也是"推理模型"比普通模型贵的原因。

装置 04 Agent 流水线模拟器 AGENT / 智能体

Agent = 模型 + 工具 + 自主规划。它不是"告诉你怎么做"，而是"自己做完交给你"。点击运行，看一个编程 Agent 接到任务后的真实工作流——注意右下角的 token 油表，这就是为什么 Agent 是吃 token 大户：

📋理解需求，把任务拆成 5 个子步骤规划
📄调用工具：读取你上传的库存 Excel工具调用 ×1
💻编写网页代码（HTML + 图表）生成
🧪调用工具：运行代码自测工具调用 ×2
⚠️发现图表数据没对齐 —— 自己定位到出错的列自查
🔁修改代码，再次运行验证通过工具调用 ×3
✅交付：一个能打开的看板网页文件完成
TOKEN 油表82,000 tokens

对比：普通聊天问一句 ≈ 1,000 token；这个 Agent 任务跑完 ≈ 82,000 token（80 倍）。这就是"龙虾套餐"和 Coding Plan 存在的原因——Agent 用户按量付费太贵，必须包月。

装置 05 Tool 与 Skill 辨析台 TOOL / SKILL / MCP

装置 04 里 Agent 反复"调用工具"，这里要分清两个容易混淆的概念：Tool（工具）决定"能做什么动作"，Skill（技能）决定"做得专业不专业"。工具是手，技能是手艺：

🔧 Tool 工具

= 手和设备 · 动作能力
Agent 能执行的具体动作：搜索网页、运行代码、读写文件、操作浏览器，每个都是一个工具。没有"文件创建"工具，模型再聪明也只能把代码打在聊天框里。

📖 Skill 技能

= 操作规范 · 经验知识
预先写好的"岗位操作手册"，Agent 动手前先读。它不增加任何新动作，只让产出从"随意发挥"变成"按行业标准交付"。

🔌 MCP 插口

= 标准接口 · 扩展工具清单
给 Agent 接入新工具的统一标准。通过 MCP 可以把日历、邮箱、数据库、内部系统变成它的新工具。

组合起来：有工具没技能 = 有手但干活糙；有技能没工具 = 懂规范但没手，啥也造不出来。下面演示 Skill 的作用（假设 Agent 已配好"文件创建"等工具）——点一个任务，看它会自动翻哪本操作手册：

幻灯片规范
📝docx 技能
文档排版规范
📈xlsx 技能
表格公式规范
🎨前端设计技能

规范内容示例：标题样式分级（标题1/2/3）/ 自动生成目录 / 表格三线表规范 / 页眉页脚与页码。
✗ 有工具 · 不带技能
通篇正文堆砌没有样式，目录是手敲的（一改内容页码全错），表格边框粗细不一。
✓ 有工具 · 装备技能
带自动目录和规范样式的 Word：改完内容右键刷新目录即可，像正式交付物。
🧭 一句话收尾：Tool 给 Agent 一双手（能不能做），Skill 给它一身手艺（做得好不好），MCP 给它一排标准插口（手还能伸多远）。三者叠加，才是一个能托付完整任务的"司机"。
装置 06 三种加油方式计算器 会员 / API / CODING PLAN

同一家 AI 公司为什么又卖会员、又卖 API、还卖 Coding Plan？因为面向三种人。拖动滑块改变你的月用量，看哪种方式最划算（以国产旗舰模型的典型价位估算，仅作示意）：

会员订阅推荐

普通用户 · 在官方 App 里聊天
¥98 /月 固定
不限按 token 计费，但只能在 App 里手动聊，不能接进程序或编程工具。轻度日常使用最省心。

API 按量推荐

开发者 · 接进自己的程序
¥2532 /月 按量
用多少付多少（此处按混合单价 ¥12/百万 token 估算）。把 AI 装进你自己的工具（比如给 control.py 加自动分析）只能走这条路。

Coding Plan推荐

程序员 · 在编程 Agent 工具里用
¥99 /月 含 ~50M token
包月送大额 token。你当前用量按 API 要 ¥2532/月，包月 ¥99 直接省 96%——这就是程序员都买它的原因。

三者不互斥：很多人上班用公司的 API 做产品，自己订个会员日常问答，写代码再开一份 Coding Plan。

SECTION 03 / 校准刻度

"是不是 Agent"不是开关，是光谱

判断标准只有一条：它是只"告诉你怎么做"，还是能"自己做完交给你"。点击三个档位对比：

完整 Agent：长时间自主执行多步任务——自己拆解目标、连续调用几十上百次工具、出错自己排查重试，直到交付最终成果，中间基本不需要你介入。
例子：Claude Code（跑一个编程任务可自主工作几十分钟）、ChatGPT Agent 模式（自己操作浏览器比价填表）、OpenClaw（龙虾）。
