AI 入门五层地图：从底层模型到你手里的产品

一条主线讲清楚：模型层、能力层、产品层、Agent 层、开发者层。读完你应该能分清 ChatGPT、DeepSeek、Claude Code、API、Token、RAG、Skill 各自在哪里。

总览 模型层 能力层 产品层 Agent 层 开发者层 Token 工具 公司地图

先记住这一句：AI 产品不是一个东西，而是一套层层叠加的系统。

大模型是底层“大脑”；产品把模型包装成你能使用的界面；工具让模型能接触外部世界；Agent 让模型从回答问题变成执行任务；API 让开发者把这些能力接进自己的系统。

模型层AI 的大脑：负责理解、推理和生成。
能力层能处理什么材料：文字、代码、图片、音频、文件。
产品层你实际打开的入口：ChatGPT、Claude、DeepSeek、Kimi。
Agent 层从“回答”升级为“办事”：规划、调用工具、检查结果。
开发者层把 AI 接入业务：API、Token 计费、RAG、Embedding、微调。

例子

第 1 层：模型层
AI 的底层大脑

输入内容，模型计算，然后输出内容。它是能力底座，但不是完整产品。

例子

GPT、Claude、Gemini、DeepSeek、Qwen、GLM、Kimi 模型。

第一层：模型层，AI 的“大脑”

大模型

用海量数据训练出来的通用模型。它能根据输入预测和生成输出。你可以把它理解为“会读、会写、会推理的语言大脑”。

Token

模型读写内容的计量单位。可以是一个汉字、一个英文词的一部分、数字、标点或空格。API 常按 token 收费。

上下文窗口

模型一次能看见的最大内容长度。窗口越大，越适合读长文档、长代码库、多轮对话。

多模态 / 推理模型

多模态是能处理文字、图像、音频、视频等；推理模型更擅长复杂步骤、数学、代码和规划。

关键边界：模型本身通常是“输入内容 → 输出内容”。能不能操作文件、生成网页、联网搜索，要看上层产品和工具。

第二层：能力层，分清“模型自带”和“工具增强”

模型自带能力

  • 理解和生成文字：解释、写作、总结、翻译。
  • 写代码和读代码：生成函数、解释报错、重构建议。
  • 图像/音频/视频理解：前提是模型支持多模态。
  • 推理和规划：把复杂问题拆成步骤。

工具增强能力

  • 联网搜索：模型调用搜索工具拿最新资料。
  • 读写文件：产品给模型文件系统或文档工具。
  • 运行代码：产品提供终端、沙箱或解释器。
  • 生成网页/文件：模型写内容，工具保存和预览。

所以 DeepSeek 网页版如果只是聊天，不代表模型不会写网页，而是当前入口没有给它“创建文件、运行代码、预览网页”的工具链。

第三层：产品层，你实际打开的入口

模型底层能力
界面聊天框、文件区
工具搜索、代码、图片
账号历史、记忆、权限
额度会员、频率、上限

ChatGPT / Claude / Gemini

通常是综合型 AI 产品：聊天、文件、图片、语音、搜索、项目、代码等能力按套餐开放。

DeepSeek / Kimi / 豆包 / 通义

也是产品入口。不同产品接的工具、文件能力、联网能力、上下文长度和额度不同。

Claude Code / Codex / Cursor

它们更像开发者工作台，重点不是聊天，而是在代码环境里执行任务。

第四层：Agent 层，从“回答问题”到“完成任务”

维度普通聊天模式Agent 模式
用户输入一个问题或一句指令一个目标或任务
AI 输出主要是一段回答过程加结果，可能包含文件、代码、报告、网页
是否规划不明显，通常一次一答会拆步骤、排顺序、决定先做什么
是否调用工具少量或没有经常调用搜索、文件、终端、浏览器、数据库
是否检查结果多数靠用户判断会运行测试、看页面、检查错误、继续修改

Agent 的工作循环

1理解目标先判断你真正想完成什么。
2制定计划拆步骤，决定顺序和风险。
3调用工具搜索、读文件、运行代码。
4执行动作写文件、改代码、生成结果。
5检查反馈看是否成功，有没有错误。
6迭代完成修正问题，直到交付。

常见 Agent 或 Agent 模式产品：Claude Code、OpenAI Codex、Cursor Agent、Devin、Replit Agent、ChatGPT Deep Research/Agent、部分企业智能体平台。普通 ChatGPT/Claude/Kimi/豆包在调用工具、多步执行时，也可以呈现 Agent 化体验。

第五层：开发者层，API、会员、RAG、微调放在哪里

会员是什么

普通用户买的是成品产品：更高额度、更强模型、文件/图片/语音/Agent 等功能。适合直接使用。

API 是什么

开发者买的是调用模型的能力：你的程序把输入发给模型，再把输出接回自己的产品。适合系统集成。

Token 计费是什么

API 通常按输入 token、输出 token、缓存、工具调用、图片/音频等资源计费。会员和 API 往往分开计费。

RAG

先从你的资料库检索相关内容，再交给模型回答。适合企业知识库、客服、内部制度问答。

Embedding

把文本转成向量，用来搜索“语义相近”的资料。RAG 系统常用它做检索。

Fine-tuning

用样例继续训练模型，让它稳定学会某种格式、风格或任务。不是所有知识库问题都需要微调。

术语索引：按所在层级理解

模型层｜Token

模型读写内容的单位。输入和输出都可能计费。

模型层｜上下文窗口

模型一次能看见的最大内容长度。

模型层｜Prompt

你给模型的任务说明、资料和输出要求。

能力层｜Tool Call

模型决定调用外部工具，例如搜索、函数、数据库、代码执行。

Agent 层｜Skill

可复用的专项说明或工具包，让 Agent 做特定任务更稳。

开发者层｜MCP

一种让 AI 连接外部工具和数据源的协议思路，可理解为 AI 工具插线板。

开发者层｜RAG

先检索资料，再让模型基于资料回答。

开发者层｜Embedding

把文本转成向量，用于语义搜索。

开发者层｜Fine-tuning

用样例继续训练模型，让模型稳定学会风格或格式。
